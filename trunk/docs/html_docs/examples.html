How to try examples
Please read the topic Tool Usage, to learn about xsd2cpp-usage, building generated source, and running the executables.
Each example directory, comes with following files:
  1. a XML-Schema file(.xsd)
  2. a valid.xml file(a valid XML file for reference)
  3. a main.cpp file


As mentioned earlier, xsd2cpp usage is:
$ xsd2cpp
Usage: xsd2cpp xsd-file [outdir]


However, for trying examples which are already organized with sensible names, and to be able to easily use the supplied main.cpp, we should run xsd2cpp within specific example subdirectory, with outdir specified as ".", like so:
$ cd simpleTypesDemo/
$ xsd2cpp simpleTypesDemo.xsd .
output path: .
=> Generating source files...
=> Generating automake/autoconf files ...


If for whatever reason you have specified a outdir other than ".", then you would need to:
  • copy the supplied main.cpp in the outdir
  • cd outdir , before building the source


For building the source in the outdir:
$ ./autogen.sh
$ make
$ make install
$ make doxygen-doc
